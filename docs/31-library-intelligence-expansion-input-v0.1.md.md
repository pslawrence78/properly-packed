# Properly Packed - Tranche 31 Library Intelligence Expansion v0.1

## Purpose

This file gives Codex supporting context for Tranche 31. It should be read before code changes begin.

The tranche is intended to make Properly Packed feel more richly intelligent at first use by expanding seeded templates, useful extras, gadget bundles and preparation tasks. It must not turn the app into a noisy generic checklist. The expansion should be curated, context-aware, explainable and safe.

## Evidence base

Two historic family packing lists have been supplied as reference material:

1. Personal Packing List - Dubai 2022.docx
2. Personal Packing List - Cruise Feb 2023.docx

These lists are evidence of historic family travel patterns. They are not gospel, not exhaustive, and not all current. Some items were age-specific to Seb at the time, such as nappies, dummies and pushchair accessories. These must not become current default suggestions for Seb, who is now older. They may be modelled only as optional young-child or toddler-context items if the existing app has a safe way to represent that context.

## Product intent reminder

Properly Packed should not be a random giant checklist.

The target product behaviour is:

- Suggest enough to avoid blank-page thinking.
- Manage the user-confirmed packing list with authority.
- Explain why something was suggested.
- Keep family-specific insight without hard-coding sensitive personal details.
- Improve through post-trip learning.
- Avoid flooding every trip with every possible item.

This tranche should increase the quality and depth of seeded intelligence while preserving user control.

## Noise-control rules

This tranche succeeds only if it improves relevance without creating clutter.

Required rules:

1. Do not add every item to every template.
2. Do not add old toddler items as current Seb defaults.
3. Do not add question-mark items as ambiguous names. Rephrase them as optional or leave them out.
4. Do not duplicate the same item under multiple slightly different names within the same owner/category context.
5. Do not hard-code sensitive home details.
6. Do not store passport numbers, insurance policy numbers, booking references, addresses or credentials.
7. Do not introduce runtime AI or external APIs.
8. Do not change the app from local-first.
9. Do not change IndexedDB schema unless absolutely necessary.
10. Do not downgrade visual quality or mobile usability.
